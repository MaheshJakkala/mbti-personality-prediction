## Overview

The MBTI framework classifies personality into 16 types across 4 binary dimensions:

| Dimension | Class 0 | Class 1 | Dataset Split |
|-----------|---------|---------|---------------|
| I vs E | Introversion | Extroversion | 77% : 23% |
| N vs S | Intuition | Sensing | 86% : 14% |
| T vs F | Thinking | Feeling | 46% : 54% |
| J vs P | Judging | Perceiving | 40% : 60% |

This project builds a **full NLP pipeline** — from raw forum posts to personality predictions — tackling a genuinely hard classification problem with heavy class imbalance and noisy labels.

## Dataset

**Source:** [Kaggle — MBTI Myers-Briggs Personality Type Dataset](https://www.kaggle.com/datasnaek/mbti-type)

- **8,675 users**, each with their last 50 forum posts (separated by `|||`)
- **16 MBTI personality type labels** with significant class imbalance
- Most common: INFP (~21%), INTJ (~15%) — Least common: ENTJ (~1.7%), ESFJ (~1.3%)

## Methodology

### Pipeline Overview

```
Raw Forum Posts (8,675 users × 50 posts each)
      │
      ▼
[Text Cleaning — preprocess.py]
  ├─ Lowercase, replace URLs with 'urlstr'
  ├─ Strip non-alphabetic characters
  ├─ Remove NLTK English stopwords
  └─ Remove MBTI type tokens  ← prevents data leakage
      │
      ▼
[Feature Engineering — Notebook 01]
  ├─ POS Tagging via spaCy: nouns, verbs, adjectives, interjections per post
  ├─ Words-per-comment, URL count, post count
  └─ TF-IDF: 10,000 features, max_df=0.3, spaCy lemma tokenizer
      │
      ▼
[Class Imbalance Handling]
  └─ SMOTETomek — oversampling (SMOTE) + Tomek link removal
      │
      ▼
[Two Modelling Strategies]
  ├── Multiclass (Notebook 02): predict all 16 types
  │     └─ Logistic Regression, Naive Bayes, Random Forest — GridSearchCV, cv=5
  └── Binary per-dimension (Notebook 03): 4 independent classifiers
        └─ Logistic Regression + XGBoost — GridSearchCV, scoring='roc_auc'
      │
      ▼
[Evaluation — evaluate.py]
  └─ Accuracy, F1 (macro + weighted), AUC-ROC, MCC, Confusion Matrix
```

### Why Binary Classification?
Framing each MBTI dimension as an independent binary problem reduces label noise and makes class imbalance more manageable — instead of 16 imbalanced classes, each classifier handles 2. It also enables targeted linguistic feature analysis per dimension.

## Honest Analysis

This section is included deliberately. Understanding *why* a model fails is as important as reporting high scores — and is what separates a serious ML practitioner from someone who just runs notebooks.

**1. The accuracy trap — NS and IE with XGBoost**
NS XGBoost shows 83.3% accuracy but sensitivity of only 4.3% for Sensing types — it predicts "Intuition" for nearly everyone. MCC of 0.005 confirms near-zero real learning. High accuracy on imbalanced data without looking at recall is meaningless.

**2. Most dimensions are near-random (AUC ~0.50)**
IE, NS, and TF all achieve AUC between 0.50–0.51, indistinguishable from a coin flip. Writing style alone — via bag-of-words features — is a weak signal for these dimensions.

**3. J vs P is the exception**
AUC 0.698 and MCC 0.276 for Logistic Regression confirms real learning. Top LR features align with known theory — Perceiving types use more open-ended, exploratory language while Judging types are more structured.

**4. SMOTETomek: a deliberate trade-off**
Resampling forces models to attempt predictions on minority classes, improving macro-F1 at the cost of precision. For this research problem this is the right call — suppressing minority class recall entirely is worse than accepting lower precision.

**5. Dataset ceiling — self-reported labels**
Users on MBTI forums know their type and likely *perform* it in their writing. This creates label circularity that no model can fully escape without a cleaner, blind-labelled dataset.

## How to Run

### 1. Clone the repo
```bash
git clone https://github.com/YOUR_USERNAME/mbti-personality-prediction.git
cd mbti-personality-prediction
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
python -m spacy download en_core_web_sm
```

### 3. Add the dataset
Download `mbti_1.csv` from [Kaggle](https://www.kaggle.com/datasnaek/mbti-type) and place it in `data/raw/`.

### 4. Run notebooks in order
```
notebooks/01 → 02 → 03
```

## Key Findings

- **Introverts dominate** (~77% of dataset) — reflects MBTI forum self-selection, not real-world base rates
- **N types overrepresented** at ~86% — single-handedly inflates accuracy for NS models; requires careful reading of sensitivity vs. specificity
- **J vs P has clear linguistic signal**: Perceiving types use exploratory, possibility-oriented language; Judging types are decisive and structured — maps well onto TF-IDF features
- **T vs F has partial signal**: words like *"feel", "heart", "love"* vs *"logic", "analysis", "argue"* are distinguishable, but class overlap is high
- **XGBoost collapses on minority classes** without sufficient data — even with SMOTETomek, gradient boosting over-optimises for majority class
- **Removing MBTI type tokens** is non-negotiable — models trained without this step learn type names, not personality signals, and report inflated fake scores
